Stage 10 / Development How the design was built — structure, stack, and system.

Technology Stack

Every tool chosen for a reason,
not out of habit.

Framework Next.js v14.2.21

App Router for clean, nested routing. Server components where content is static. The route group structure /(public) and /app separates the marketing site from the member platform cleanly.

Language TypeScript v5

Type safety across props and data structures. Catches errors at build time rather than runtime — important for a membership-gated app where component props are complex.

Styling Tailwind CSS v3

Utility classes for layout and spacing. The custom design system (colours, typography) lives in globals.css as CSS custom properties — not in Tailwind config — keeping tokens framework-agnostic.

Runtime React v18

Used for interactive client components — the theme toggle (ThemeContext), auth state (AuthContext), the custom cursor (CustomCursor), and the FAQ accordion.

Architecture Decisions

Key structural choices made
during the build.

Routing

Route groups for separation

Next.js route groups — (public) and app — separate the marketing site from the member platform at the folder level. Each has its own layout.tsx defining a completely different shell: Navbar + Footer for marketing, AppSidebar for the platform.
A member inside the platform should never see the marketing nav. A visitor should never see the platform sidebar. The separation is structural, not conditional — no logic needed to show or hide navigation.

Theming

CSS tokens via data-theme

Colours are CSS custom properties defined as raw RGB values (e.g. 10 10 10). Consumed as rgb(var(--c-bg)). Switching themes changes a single data-theme attribute on the html element, which remaps all tokens instantly via CSS cascade.
No component needs to know which theme is active. Every element automatically re-renders in the correct palette. Adding a third theme (e.g. a high-contrast mode) would require only a new CSS block in globals.css — nothing else changes.

Auth

AuthGuard component

A single AuthGuard.tsx component wraps the /app layout. It checks auth state from AuthContext and redirects unauthenticated users to /sign-in. All platform pages are protected automatically without adding auth logic to each page.
Centralising auth at the layout level means no individual page can accidentally be left unprotected. New platform pages added under /app are automatically gated — no extra step required.

Cursor

Custom cursor as client component

The system cursor is hidden via CSS (cursor: none) in globals.css. A client component CustomCursor.tsx tracks mouse position with a mousemove event listener and renders a small dot that follows the cursor. It scales up on interactive element hover.
A custom cursor is a detail that aligns with the brand's attention to craft. It's implemented as an isolated client component so it doesn't force the rest of the app into client rendering.

Fonts

Next.js font optimisation

Cormorant Garamond and Inter are loaded via next/font with display: swap. Fonts are self-hosted and inlined into the page — no external request to Google Fonts at runtime. Defined as CSS custom properties (--font-cormorant, --font-inter) for use across the token system.
Self-hosting fonts removes a third-party network dependency and eliminates layout shift. Typography loads instantly with the page rather than after it.
