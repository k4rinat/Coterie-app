Stage 05 / Design Iterations What changed, what stayed, and why each decision was made.

Version History

The design went through three clear phases.

Version 01

First build — structure without style

The first version was about getting the page structure right. All sections were in place, the navigation worked, the pages existed. But the visual design used system defaults — Inter Regular, standard weights, hardcoded hex colours. It looked like a placeholder, not a product. The layout was right. Everything else needed work.

All pages built Navigation functional System fonts (Inter Regular) Colours hardcoded No dark mode No visual rhythm
Version 02

Typography and token overhaul

The biggest shift. Cormorant Garamond 300 replaced all headings. Inter was locked to weight 300 for body and 500 for labels only. All colours were moved out of components and into CSS custom properties. This was also when the dark mode architecture was designed — not implemented yet, but the token system was built to support it from day one.

Cormorant Garamond 300 Inter 300 body / 500 labels CSS token system Dark mode architecture Section spacing rhythm Hairline CTAs
Version 03

Refinement — removing what didn't belong

The third pass was about subtraction. The hero had too many elements — bullet points, a supporting image, a background grid pattern. All removed. The apply form had 12 fields. Reduced to 4. The membership section had a highlighted "recommended" tier — removed. Each removal made the page closer to the reference. The rule became: if you can remove it and the page is still clear, remove it.

Hero stripped to headline + CTA only Form: 12 fields → 4 Tier cards equalised Mobile nav added Dark mode live Footer simplified

What never changed

Some decisions were made once and held.

Page structure

The homepage section order — Hero, Pillars, Disciplines, Membership, Quote, CTA — was set in the wireframe stage and never moved. The logic was right the first time.

No accent colour

From the first reference analysis, the decision was monochrome. It was never revisited. Introducing colour at any stage would have changed what the site communicates.

Single conversion goal

Every page points to the application. There's no newsletter, no download, no secondary CTA. That decision was made at the brief stage and never changed.
