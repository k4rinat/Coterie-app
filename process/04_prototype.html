Design Tokens

globals.css — all decisions in one place.

globals.css
/* Single source of truth for all colour */ :root { /* backgrounds */ --c-bg: #ffffff; --c-surface: #fcfcfa; --c-elevated: #f4f4f2; --c-rule: #dcdcda; /* text */ --c-muted: #b4b4b2; --c-mid: #787876; --c-ink: #0e0e0e; /* typography */ --f-display: 'Cormorant Garamond', serif; --f-body: 'Inter', sans-serif; --fw-light: 300; --fw-mid: 500; } [data-theme='dark'] { --c-bg: #0c0c0c; --c-surface: #181818; --c-elevated: #282828; --c-rule: #404042; --c-muted: #8c8c91; --c-mid: #bebec3; --c-ink: #f2f1ef; }

Named by purpose, not colour

--c-ink instead of --black. When dark mode flips it to near-white, the token name still makes sense. No confusion, no rewrites.

Dark mode = one override block

data-theme="dark" just redefines the same variables. Every component updates automatically — no duplicate CSS, no JavaScript required.

Tailwind maps to the tokens

tailwind.config.js points text-ink, bg-surface, border-rule etc. at these CSS variables — so utility classes always match the active theme.

One file to change the whole site

Every colour decision lives here. Nothing hardcoded in any component. Change --c-ink and every heading, button, and rule updates instantly.

Component Architecture

Section component — one building block for every page.

<section className="py-24 w-full"> Full page width · vertical padding from design token
<div className="max-w-4xl mx-auto px-6"> Max-width 896px · horizontally centred · consistent side padding
<header> (optional) Stage label in small caps + section heading in Cormorant — only when needed
{children} Any content — grids, cards, text, lists. Section never dictates this.
